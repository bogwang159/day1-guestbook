## 1. 가장 막혔던 지점
- EC2 SSH 접속 문제: PEM 파일 권한 문제, 보안 그룹 설정 오류, IP 변경 등으로 접속 실패
- Docker 빌드 문제: Backend JAR 파일이 없어서 `COPY build/libs/*.jar app.jar` 단계에서 실패
- GitHub Actions 자동 배포 오류: Docker Hub 로그인 실패로 배포 중단

## 2. 이해가 부족하다고 느낀 부분
- GitHub Actions의 SSH 자동 배포 과정과 Secrets 사용 방법
- Docker 이미지 빌드 캐시와 레이어 구조 이해
- EC2 보안 그룹과 퍼블릭 IP 관리

## 3. 팀 프로젝트 전에 보완하고 싶은 기술
- CI/CD 자동화 설정 경험: GitHub Actions, Docker Hub, EC2 배포
- 프론트엔드와 백엔드 Docker 빌드 과정 완전 이해
- AWS EC2, 보안 그룹, 퍼블릭 IP 관리 경험

## 4. 혼자 진행하며 느낀 점
- 혼자서 배포 환경을 설정하고 문제를 해결하면서 DevOps의 중요성을 크게 느꼈음
- 에러 로그를 읽고 원인을 추적하는 능력이 많이 향상됨
- 자동화된 배포 환경이 없으면 배포 과정이 얼마나 번거로운지 체감
