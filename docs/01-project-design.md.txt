# 프로젝트 설계

## 1. 서비스 설명
사용자가 간단한 다짐이나 한 줄 메시지를 작성하고 공유할 수 있는 방명록 서비스이다.

## 2. 전체 흐름
사용자가 브라우저에서 다짐 내용을 입력하면 프론트엔드가 백엔드 API로 요청을 전송한다.  
백엔드는 요청 데이터를 검증한 후 MySQL 데이터베이스에 저장하고,  
저장된 결과를 다시 프론트엔드로 반환한다.  
프론트엔드는 응답 데이터를 화면에 목록 형태로 렌더링한다.

## 3. 프론트엔드 역할
- 사용자 입력 UI 제공
- 다짐 목록 조회 화면 렌더링
- 백엔드 API 호출 (조회, 등록)
- 서버로부터 받은 데이터를 화면에 표시

## 4. 백엔드 역할
- REST API 제공
- 다짐 데이터의 생성 및 조회 처리
- 데이터베이스와의 연동
- 비즈니스 로직 처리 및 데이터 검증

## 5. 배포 구조 요약
- 로컬 환경:  
  프론트엔드, 백엔드, 데이터베이스를 각각 로컬에서 실행하며 개발

- 배포 환경:  
  AWS EC2 인스턴스 위에서 Docker와 Docker Compose를 사용하여  
  프론트엔드, 백엔드, MySQL을 각각 컨테이너로 실행
